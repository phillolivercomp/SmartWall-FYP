Smartwall

This piece of software aims to provide you with enough information to fully monitor selected devices on a users network alongside securing them against external threats.

Technology used

The web interface has been achieved using Javascript with JQuery. Other API's have been used in conjunction which are needed for full functionalty. Alongside this, lua is used to talk between the web server and the other elements of the router. Most of the capture is achieved using Python scripting. The SQL database/s are SQLite based

How to Use

Smartwall is split up into a different parts with the overall goal of providing clarity for exactly what IoT (and other) devices are connecting to with the ability to stop them accessing what you do not want them to. This comes with the ability for the user to have massive control over their devices in a simple and efficient way.

Control Center

This is the first page a user will use to evaluate their devices security. The configured devices appear here with a history of the amount of traffic they have been sending and receiving over time. Beneath this are two submenus being the connections page and the monitoring page.

Connections page is the more strict way to access device control. Firstly there is a box showing some statistics about what the device is connecting to both on the local (LAN) and wider network (WAN). From here, we can click limit connections to generate iptable rules. These rules are viewable in browser and can be added to or removed with the controls on the page. If you want to stop limiting connections, this can be done by hitting unlimit connections at the bottom of the page.

Monitoring page is the soft security solution. A profile of behaviour is generated at this point based on what options you have selected. Once you click start monitoring, all connections from this point on will be checked against this profile report and based on the rules set will warn you of behaviour that is unexpected. Each rule does the following:

  • IP Addresses - Logs any new IP addresses the devices has connected to.
  • Ports - Logs new services / ports the devices uses.
  • Data in/out ratio - Checks the amount of data sent in and out of the device each hour and compartive to previous ratios will flag big changes in these numbers
  • Max data - Takes max data within an hour previously and if the device exceeds this by a large amount it will be logged

Connections tab

The connections tab provides is simply an interface between the web UI and the underlying database. This database stores every connection the device you have selected has made since monitoring began. Hostnames will attempt to be resolved against found IP addresses but in the case where the name is either not found or not easily identifiable, this name can be changed by the user.

By clicking an entry in this take, a breakdown page opens of the entirity of connections made between the device and this IP address. This includes the amount of data sent on each port and data draw from a geolocation service.

Configuration

Configuration page allows the user to make changes to the way Smartwall performs. This includes specifying the devices that you want to monitor. The MAC addresses in the entry table should automatically populate with devices on the network alongside their IP addresses if available. Adding or removing devices is simple with devices added being monitored immediately as of being saved.
